Technologies Used:

  • Frontend: Angular framework for building the user interface.
  • Backend: Express.js for building the RESTful API.
  • Database: MongoDB for storing phone and customer data.

Key Features:

  • Phone Catalog: Users can view a catalog of available phones, each with details like name, price, manufacturer, and image.
  • Customer Management: The system allows the management of customer information, including personal details, home and shipping addresses, and purchase history.
  • Customer Purchase History: Each customer has a purchase history that records the phones they have bought, including details like phone ID, name, and price.
  • Seeding Sample Data: Seed routes are provided to populate the database with sample phone and customer data for testing and development purposes.
  • Routing: Angular routing is used to navigate between different pages of the application, such as the home page, phone details page, and customer details page.

Implementation Details:

  • API Endpoints: Express.js is used to define API routes for fetching phone and customer data, as well as routes for seeding sample data into the database.
  • Data Models: Mongoose schemas are defined for the phone and customer data models, ensuring consistency and validation of data.
  • Angular Components: Angular components are created for different sections of the application, such as the home page, phone details page, customer list page, and customer details page.
  • Routing: Angular routing is configured to handle navigation between different components/pages of the application based on user actions.
